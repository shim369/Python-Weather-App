# Python Weather App 要件定義書

## 1. 業務要件

### システム概要

個人利用を目的としたGUIベースの天気情報取得アプリケーションを開発する。

ユーザーは都市名を入力し、外部天気APIから現在の天気情報を取得・表示できる。

また、取得した天気情報をSQLiteへ保存し、過去の検索履歴を参照できる。

### 想定利用者

* 個人ユーザー（1名）

### ユースケース

#### 一般ユーザー

* アプリケーションを起動する
* 都市名を入力する
* 天気情報を取得する
* 現在の天気情報を確認する
* 過去の検索履歴を確認する
* エラーメッセージを確認する

---

## 2. 機能要件

### アプリケーション種別

* デスクトップGUIアプリケーション
* Tkinterを使用する
* Webアプリケーションではない

### 機能一覧

#### 天気取得機能

* 都市名入力
* 天気API呼び出し
* 天気情報取得
* APIレスポンス解析

#### 天気表示機能

* 都市名表示
* 天気説明表示
* 気温表示
* 湿度表示
* 取得日時表示

#### 検索履歴機能

* 検索結果保存
* 検索履歴取得
* 検索履歴表示

#### データ管理

* Weather(dataclass)によるデータ管理
* SQLiteによる永続化

#### エラー処理

* 通信エラー表示
* APIエラー表示
* 入力値エラー表示
* DBエラー表示

#### ログ出力

* エラーログ出力
* 操作ログ出力

### 外部システム連携

#### 天気API

* OpenWeather API
* HTTP通信(httpx)

---

## 3. 非機能要件

### 開発環境

* Python 3.13
* uv
* VSCode
* GitHub
* GitHub Actions

### アプリケーション構成

* ローカル実行型GUIアプリケーション
* データ保存先はSQLite

### キャパシティプランニング

#### 想定ユーザー数

* 1人

#### 想定検索履歴件数

* 10,000件程度

#### SQLite容量

* 数十MB以内

### 性能要件

* 天気取得：5秒以内
* 検索履歴表示：1秒以内
* SQLite保存：1秒以内

### 信頼性

* API通信失敗時にエラーメッセージを表示する
* SQLite障害時にエラーメッセージを表示する
* 不正なAPIレスポンスに対応する

### 保守性

* 型ヒントを使用する
* mypyで静的解析を実施する
* Ruffでコード品質を維持する
* pytestで自動テストを実施する
* GitHub ActionsでCIを実施する

### セキュリティ要件

* ユーザー入力のバリデーションを行う
* APIキーをソースコードへ直接記載しない
* 環境変数でAPIキーを管理する
* SQLインジェクション対策を行う

### ログ要件

* エラーログを出力する
* API通信ログを記録する
* 操作ログを記録する
